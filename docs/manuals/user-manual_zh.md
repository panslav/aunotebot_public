# 用户指南

本指南说明如何在 Telegram 中使用 AUNotebot。

维护说明（v0.25 音频大小限制澄清）：本机器人对 Telegram 上传音频采用 Telegram Bot API 下载大小限制。

## 1. 开始使用
1. 打开机器人聊天窗口。
2. 发送 `/start`。
3. `/start` 会始终重新加载并重新应用你已保存的设置，尤其是“语言”和“底部菜单可见性”。
4. 你将看到：
   - 欢迎消息：
     - `欢迎使用 AUNotebot。`
     - `在生活发生的地方记录你的笔记！`
     - `📝🗣️🎥🌃➡️🧠✅`
     - `📝 记录你的想法、思考和任务。`
     - `🛡️ 再也不要遗忘任何事情。`
     - `1️⃣ 给我发文字、语音、图片、圆形视频或普通视频。`
     - `2️⃣ 我会保存它，并创建笔记或任务。我的 AI 会分析并总结你的内容。`
     - `3️⃣ 保持为笔记，或者把它变成可分配的任务！`
     - `⚖️ 使用本机器人即表示你同意条款与条件（点击 More -> Legal）。`
     - `❓ 点击 More -> Help 查看用户手册。`
     - `✉️ 想法或反馈：@aunotebot_support`
   - 欢迎消息最后一行显示当前版本
   - 内联快捷入口与计数：
     - 第 1 行：`Find`
     - 第 2 行：`Tasks`、`Notes`
     - 第 3 行：`Reminders`、`Drafts`
     - 第 4 行：`More`
   - Telegram 汉堡菜单中的快捷命令

公开条款文档：
- [法律文档仓库](https://github.com/panslav/aunotebot_public/tree/main/docs/legal)
- 打开 `More -> Legal` 可按当前 UI 语言打开条款页面。
- OpenAI 使用政策（AI 功能基线）：
  - https://openai.com/en-GB/policies/usage-policies/
  - 如果请求违反这些政策，AI 功能可能会被限制或禁用。
- 管理员跟踪说明：
  - 机器人会保存操作审计记录（`moment`、`action_type`）以及分享关系边（`source`、`target`、`action_type`），用于运营与图分析。

公开用户手册：
- [公开手册仓库](https://github.com/panslav/aunotebot_public/tree/main/docs/manuals)
- 打开 `More -> Help` 可按当前 UI 语言打开对应手册页面。

## 2. 语言
1. 首次 `/start` 时，机器人会尝试根据 Telegram 语言环境识别你的语言。
2. 默认回退语言为英语。
3. 如需手动切换语言：
   - 打开 `More -> Settings -> Language`
   - 选择：`English`、`Español`、`Français`、`Português`、`Русский`、`Українська`、`简体中文`
4. 选择后，所有核心 UI 标签和按钮都会切换为所选语言。

## 2a. 国家
1. 打开 `More -> Settings -> Country`。
2. 在带有国旗的分页国家列表中选择你的国家。
3. 机器人不会自动填写该值；只有你明确选择后才会保存。
4. 此信息用于 AI 使用前的税务与计费合规。

## 2b. AI 积分与 Telegram Stars
1. AI 功能会消耗你的积分余额。
2. 默认计费模型：
   - `1 积分 = 1 Telegram Star`
   - 首次启动时赠送 `50 免费积分`
   - 每天本地时间 `00:00` 赠送 `5 免费积分`
   - 文本转语音：`每分钟 1.5 积分`
   - 音频解析：`每分钟 0.25 积分`
   - 图片 OCR：`每张 0.1 积分`
   - 文本/链接分析：`每次 0.05 积分`
3. 打开 `More -> Points` 可以：
   - 查看当前余额
   - 查看最近 5 条积分记录
   - 打开 `完整积分记录`
   - 查看当前 AI 计费
   - 使用 Telegram Stars 充值
4. 在使用 AI 功能或购买积分前，你必须先设置 `Country`。
5. 在机器人内购买积分时，Telegram Stars 充值直接按 `XTR` 计价，不会因国家而额外加收机器人侧 VAT。
6. 如果余额不足：
   - 该次 AI 功能会被阻止
   - 机器人会显示提示
   - 非 AI 功能仍然可以继续使用

## 3. 创建笔记
1. 直接给机器人发送文本。
2. 机器人会显示草稿预览。
3. 点击 `📝 Save as Note`。
4. 随后会打开笔记列表。

## 4. 创建任务
方式 A：
1. 发送文本。
2. 在草稿预览中点击 `🟧 Save as Task`。
3. 随后会打开任务列表。

方式 B：
1. 打开任意笔记。
2. 点击 `⬛ Make Task`。

## 5. 转发消息
1. 将一条或多条消息转发给机器人。
2. 机器人会把时间相近的转发消息合并成一个草稿。
   - 默认合并时间为 `30` 秒。
   - 你可以在 `More -> Settings -> Multiple message merge time` 中修改。
3. 如果你在转发后紧接着发送普通文本，机器人会把它追加为：
   - `💬 Forwarded with the comment:`
   - `<你的文本>`
   并放在来源/页脚之前。
4. 在笔记/任务视图中，转发来源会显示完整 URL：
   - `Source: https://...`
5. 可选：在保存前发送仅包含话题标签的文本（例如 `#work #idea`）以添加标签。
6. 打开草稿后可以使用：
   - 第 1 行：`📝 Save as Note`、`🟧 Save as Task`
   - 第 2 行：`✏️ Edit`、`❌ Cancel`
7. 如果所有草稿都已保存或处理完，再打开 `Drafts` 会返回到主页选择界面。

## 6. 语音和视频
1. 发送语音消息、音频文件、视频或圆形视频。
2. 当开始解析音频/视频时，机器人会立即显示处理状态消息。
3. 机器人会转写并生成草稿。
4. 点击 `📝 Save as Note` 或 `🟧 Save as Task`。
5. 大于 `20 MB` 的 Telegram 语音、音频和视频文件会在处理前被拒绝，因为当前机器人运行方式下 Telegram Bot API 无法下载更大的上传文件。
6. 如果你的录音或视频大于 `20 MB`，请手动拆分或压缩后分段发送。

## 7. 图片和截图
1. 发送或转发图片/截图。
2. 机器人会执行 OCR，并总结可见的关键文字/内容。
3. 系统会创建草稿，然后你可以将其保存为笔记或任务。

## 8. 浏览笔记和任务
- 使用 Telegram 命令菜单（汉堡菜单）：
  - `🏠 Start`
  - `🔎 Find`
  - `📝 Notes`
  - `🗂️ Drafts`
  - `🟧 Tasks`
  - `⏰ Reminders`
  - `📚 More`
- 如果启用了底部菜单（`More -> Settings -> Show bottom menu`）：
  - 第 1 行：`📝 Notes` + 下一行 + `(N)`，`⏰ Reminders` + 下一行 + `(N)`，`🟧 Tasks` + 下一行 + `(N)`
  - 第 2 行：`🗂️ Drafts` + 下一行 + `(N)`，`🔎 Find`，`📚 More`
  - 启用后，底部菜单会一直显示，直到你在设置中关闭它
  - 这些按钮与汉堡菜单中的入口打开相同界面。
- 也可使用欢迎界面上的内联入口：
  - 第 1 行：`📝 Notes (N)`、`🟧 Tasks (N)`
  - 第 2 行：`🗂️ Drafts (N)`、`⏰ Reminders (N)`
  - 第 3 行：`🔎 Find`、`📚 More`

## 9. 查找（合并搜索 + Recall）
1. 点击 `🔎 Find`（欢迎界面、汉堡菜单或 `/find`）。
2. 机器人会进入交互式 Find 模式并显示：
   - `🔎 你可以问我任何问题！我的 AI 会为你查找并总结信息。`
3. 发送任意自由文本查询或问题。
4. 机器人会同时进行：
   - 对 Notes 和 Tasks（包括已归档项）进行确定性排序检索
   - 识别查询词的不同词形变化（例如名词词尾变化）
   - 对相关结果进行 AI 总结
5. 当找到项目时，回复格式为：
   - 总结文本
   - `Here are the most relevant items for your request:`
   - 相关项目按钮（最相关的排在最前）
   - `⬅️ Back`
6. 如果你的 Find 查询是精确项目 ID（例如 `219` 或 `#219`），并且该项目存在，那么它会固定为结果第 1 条。
7. 点击 `⬅️ Back` 可退出 Find 模式。
8. `⬅️ Back` 会返回到进入 Find 之前所在的界面。

## 9a. 提醒建议
1. 如果新创建的笔记或任务中包含明显的日期/时间短语，机器人会自动建议创建提醒。
2. 如果该项目仍是普通笔记，而你选择 `Set reminder`，机器人会先把它转换为任务，然后再安排提醒。
3. 创建提醒后，机器人会用你的语言确认提醒时间。

## 10. 笔记和任务操作

### 笔记操作
- `✏️ Edit`
- `⬛ Task`
- `❌ Remove`
- `📚 More`
- `⬅️ Back`
- `📚 More` 打开：
  - `🏷️ Tags`
  - `🔊 Make Voice`
  - `🔀 Merge`
  - `📤 Share`
  - `📌 Pin` / `📌 Unpin`

### 任务操作
- `✏️ Edit`
- `✅ Done` 或 `🔄 Re-do`
- `⏰ Reminder`
- `🔁 Make Note`
- `❌ Remove`
- `📚 More`
- `⬅️ Back`
- `📚 More` 打开：
  - `🏷️ Tags`
  - `🔊 Make Voice`
  - `🔀 Merge`
  - `👥 Assign` 或 `👥 Re-assign`
  - `📌 Pin` / `📌 Unpin`

### 通过 Contacts 分配任务
1. 打开一个任务。
2. 点击 `👥 Assign`（或 `👥 Re-assign`）。
3. 机器人会显示：
   - 你已保存的联系人按钮
   - 手动输入选项（`@handle` 或手机号）
4. 如果你输入新的有效 `@handle`/手机号，它会自动保存到 Contacts。
5. 如果接收人已经启动过机器人，任务会被发送并与该用户建立关联。
6. 你可以在分配选择器里使用 `↩️ Unassign` 取消当前分配。

### 通过 Contacts 分享笔记
1. 打开普通笔记。
2. 点击 `📤 Share`。
3. 机器人会显示：
   - 你已保存的联系人按钮
   - 手动输入选项（`@handle` 或手机号）
4. 如果你输入新的有效 `@handle`/手机号，它会自动保存到 Contacts。
5. 如果接收人已经启动过机器人，笔记会被分享给对方，并发送到对方聊天中（包括附件）。

### 生成语音（文字转语音）
1. 打开任意笔记或任务。
2. 点击 `🔊 Make Voice`。
3. 机器人会立即显示系统通知，说明文字转语音已开始。
4. 机器人使用 OpenAI（`onyx` 音色）生成语音消息，发送到聊天中，并把它附加到该项目。

## 11. 合并项目（笔记/任务）
1. 打开一个项目。
2. 点击 `🔗 Merge`。
3. 机器人会显示最多 5 个最近打开的同类型项目。
4. 选择目标项目，或通过文字发送目标项目 ID。
5. 源项目的文本会追加到目标项目文本末尾。
6. 源项目会被移除，然后打开目标项目。

## 11.1 置顶项目
1. 打开笔记或任务。
2. 点击 `📚 More`。
3. 点击 `📌 Pin`，该项目会始终位于列表顶部。
4. 点击 `📌 Unpin`，项目恢复正常时间排序。

## 12. 标签
1. 打开一个笔记。
2. 点击 `🏷️ Tags`。
3. 点击某个标签可打开带有该标签的笔记列表。
4. 要添加标签，在消息中发送话题标签（例如 `#project #urgent`）。
5. 笔记文本中识别出的人名会自动作为首字母大写的标签加入。

## 13. 编辑笔记
1. 打开笔记/任务。
2. 点击 `✏️ Edit`。
3. 按提示发送完整的更新文本。
4. 机器人会保存修改，并在页脚添加 `last edit` 时间戳。

## 14. 删除或归档
1. 打开笔记/任务。
2. 点击 `❌ Remove`。
3. 选择：
   - `🗑️ Delete`
   - `🗄️ Archive`
   - `⬅️ Back`
4. 如果该笔记/任务在聊天中显示过媒体附件，那么删除/归档后，这些附件消息也会从当前界面移除。

## 15. 完成任务与评论
1. 打开任务。
2. 点击 `✅ Done`。
3. 输入可选评论，或点击 `✅ No comments`。
4. 结果遵循你的 Settings 选项：
   - Delete
   - Archive
   - Keep

## 16. 提醒
1. 打开任务。
2. 点击 `⏰ Reminder`。
3. 发送自由格式的日期/时间，例如：
   - `明天同一时间`
   - `2 小时后`
   - `2026-02-20 14:30`

## 17. 设置
打开 `⚙️ Settings` 可配置：
- `🌐 Language`（第一项）
  - English / Русский / Українська / Español / Português / Français / 简体中文
- `🌐 Timezone`
  - 点击 `Timezone`
  - 回复城市、时区代码，或你当前时间的文本描述
  - 机器人会使用 OpenAI 推断并保存你的时区
- `Mark as done`
  - Delete / Archive / Keep
- `Copy text for edits`
  - Copy note text / Do nothing
- `🗓️ Daily task digest`
  - `Set time`（例如 `每天早上 8 点`、`21:30`、`明天 7 点`）
  - 保存时间也会自动启用每日摘要
  - `Enable` / `Disable`
  - 摘要会在用户本地时间每天发送一次
  - 摘要包含：
    - `Reminders`（或 `No reminders until next digest`）
    - `Tasks`
    - `Notes added since the last digest`
  - 所有条目都是链接；点击可在机器人中打开对应笔记/任务
  - 摘要消息包含快捷按钮：`⏰ Reminders`、`🟧 Tasks`
  - 设置摘要页与详情页中的状态会保持同步
- `🌙 Evening summary`
  - `Set time`（与晨间摘要使用同一个自由时间解析器）
  - 保存时间也会自动启用晚间摘要
  - `Enable` / `Disable`
  - 晚间摘要包含：
    - `Tasks -> 1.1 今天我完成的任务`
    - `Tasks -> 1.2 今天我创建的任务`
    - `今天新增的笔记`
  - 在摘要后，机器人会提问：
    - `你觉得今天过得怎么样？`
    - 针对该条提示回复文字/语音/视频
    - 机器人会把你的回复保存为一个带有 `daily` 标签的笔记
- `⏱️ Multiple message merge time`
  - 点击该按钮
  - 发送秒数（例如 `30`、`45`、`120`）
  - 在该时间间隔内发送的消息会合并为一个草稿
  - 如果间隔达到 `30` 秒或以上（默认设置），则会创建新草稿
- `📱 Show bottom menu`
  - 状态使用绿/红圆点显示
  - 默认：启用
  - 启用后，该用户将看到持续显示的底部菜单键盘
  - 关闭后，该用户的底部菜单键盘会被隐藏
  - 修改此设置是静默的（不会出现额外系统通知）

联系人管理位于 `More -> Contacts, Invite`：
- 查看所有已保存的分配联系人
- `➕ Add contact`（输入 `@handle` 或手机号）
- 打开联系人 -> `Edit` / `Delete`
- 编辑后会更新在任务分配建议中使用的联系人值

## 18. 使用提示
- 如果列表很长，请使用分页按钮。
- 使用命令菜单（`/start`、`Notes`、`Tasks` 等）可以快速导航。
- `Archive` 会将项目从普通笔记/任务列表中隐藏，但不会删除数据。

## 19. 如果看起来像是客户端问题
如果操作看起来卡住了（例如点击按钮后发送了你的消息，但机器人没有响应），请尝试：
- 发送 `/start`
- 点击欢迎界面上的任意内联按钮
- 点击或发送 `⬅️ Back`（或 `/back`）
- 关闭并重新打开 Telegram 应用
- 如果仍然无效，清空聊天记录并重新打开机器人聊天

## 20. 如何更好地报告 Bug
报告 bug 时，请附上：
- 你点击的准确按钮/操作
- 准确的日期、时间和时区
- 截图或简短录屏
- 机器人的准确响应（或说明没有响应）
- 当其他按钮失效时，`/start` 是否仍然可用
